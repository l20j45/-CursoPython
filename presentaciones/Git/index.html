nosotros para poder trabajar con los archivos no basta con inicir el repositorio, tambien se necesita agregarlos al area de visualizacion para agregarlos al area de stagin o de observacion podemos usar el codigo git add

Hasta aqui el uso inicial de git esta cubierto
Que pasa si nosotros tenemos que hacer un cambio en el codigo pero no queremos afectar la linea inicial del programa?
para esto usamos
BRANCHS
tambien esto nos ayuda muchisimo al trabajo colaborativo
para crear la branch podemos hacer de esta forma 

                        git branch (nombre de la rama)
para cambiarnos de rama podemos usar el codigo checkout 

                        git checkout (nombre de la rama)
con esto podemos trabajar partes en diferentes ramas sin afectar el codigo inicial y de esta forma probar opciones o probar nuevas acciones en el codigo sin sentir miedo a que no funcione
cuando queramos temrinar con la rama y agregarlos a la rama principal podemos usar el codigo merge que seria unir las ramas
1 cambiar a la rama a donde vamos a unir el codigos
unir el codigos 

    git merge (branch)
luego guardamos el cambio con un commit o un truco mayor podemos juntar el codigo de git add junto co el codigo commit de esta manera 

                        git commit -am "comentario"
podemos ver los cambios entre versiones de los commits o entre las branchs con el comando 

                        git diff
para ver los cambios podemos usar git log para lo que estamos haciendo 

                        git log
para ver los cambios actuales usamos git status 

                        git status

es mas sencillo trabajarlo asi por cuestiones de practicidad y para tomar menos tiempo lo haremos de esta forma
lo siguiente es que con el codigo de 

                        git pull origin (nombre de la rama)
si queremos mandar los cambios a internet tenemos que hacer todo el commit, luego agregar todo y de tal forma mandamos todo a la rama principal, siempre primero lo mandas a main o master depende de, y luego usamos 

                    git pull origin (nombre de la rama)

                        git push origin (nombre de la rama)
